# سیستم طراحی ابر سفید

تنها منبع حقیقتِ رنگ و تایپوگرافی، بلوک `@theme` در [`app/globals.css`](app/globals.css) است.
هرجای دیگری رنگ خام (`#A8C4D4`) یا کلاس پیش‌فرض تیلویند (`gray-500`, `blue-400`) ننویسید —
اگر روزی پالت عوض شود، فقط همان یک فایل باید تغییر کند.

## پالت

| توکن | کد | نقش |
|---|---|---|
| `bg` | `#FAF8F4` | پس‌زمینهٔ اصلی — سفید شکری |
| `surface` | `#F0EBE3` | کارت‌ها و سطح دوم — بژ روشن |
| `line` | `#E3DCD2` | خطوط مویی و حاشیه‌ها |
| `ink` | `#1B3A50` | متن اصلی — سرمه‌ای عمیق |
| `muted` | `#456A80` | متن ثانویه |
| `subtle` | `#6E8C9E` | فقط متن درشت |
| `sky` | `#A8C4D4` | پُرکنندهٔ آبی — دکمه، بج، ابرهای صفحهٔ ورودی |
| `sky-hover` | `#94B5C8` | هاورِ همان پُرکننده |
| `sky-deep` | `#7BA0B5` | فقط تزئینی — نشان، ابرهای پس‌زمینه |
| `sky-text` | `#3F6880` | آبی در نقش متن — لینک |

## هیچ مشکی‌ای در این سایت نیست

متن‌ها سرمه‌ای‌اند، نه مشکی و نه خاکستریِ خنثی. دلیلش فقط سلیقه نیست:

**۱. جوهر واقعی هم مشکی نیست.** جوهرهای کلاسیک همیشه ته‌رنگ آبی یا قهوه‌ای
داشته‌اند؛ مشکیِ خالص چیزی است که چاپگر لیزری تولید می‌کند، نه قلم.

**۲. یک خانوادهٔ رنگی به‌جای دو تا.** پیش از این متن‌ها قهوه‌ای-خاکستری
(`#3A3632`) بودند و لهجه آبی — دو خانوادهٔ جدا که کنار هم می‌نشستند. حالا
متن، لینک و لهجه همه از یک طیف‌اند و صفحه یکدست‌تر است.

**۳. روی زمینهٔ گرم، آبیِ سرد نرم‌تر می‌نشیند.** مشکی روی کرم لبهٔ تیزی
می‌سازد که با حال‌وهوای آرام استودیو نمی‌خواند.

سایه‌ها هم همین ته‌رنگ را دارند (`rgb(27 58 80)`) نه خاکستری خنثی — سایهٔ
خاکستری روی زمینهٔ کرم کدر و دودی می‌افتد.

> **مهم:** تیره‌تر شدن رنگ متن، آبی‌بودنش را از بین نمی‌برد. اگر روزی خواستید
> متن را پررنگ‌تر کنید، اشباع را نگه دارید و فقط روشنایی را کم کنید. اندازه‌گیری
> نشان داد آبی‌تر کردن `ink` هیچ هزینه‌ای در کنتراست ندارد.

## چرا چهار آبی؟

پرامپت اولیه دو آبی داشت (`#A8C4D4` و `#7BA0B5`) و می‌گفت برای «دکمه، لینک، جزئیات».
اندازه‌گیری کنتراست نشان داد این کار متن را ناخوانا می‌کند:

| ترکیب | کنتراست | حکم |
|---|---|---|
| متن سفید روی `#A8C4D4` | ۱٫۸:۱ | ✗ فاجعه |
| `#A8C4D4` به‌عنوان متنِ لینک روی `bg` | ۱٫۷:۱ | ✗ تقریباً نامرئی |
| `#7BA0B5` به‌عنوان متنِ لینک روی `bg` | ۲٫۶:۱ | ✗ مردود |

حداقل استاندارد WCAG 2.1 AA برای متن معمولی **۴٫۵:۱** است.

راه‌حل، عوض کردن حال‌وهوا نبود؛ **تفکیک نقش** بود. آبیِ روشن و محو همچنان آبیِ
غالبِ بصری سایت است، ولی هرگز خودش متن نمی‌شود:

- `sky` / `sky-hover` → **فقط پُرکننده**. متن روی آن همیشه `ink` است.
- `sky-deep` → **فقط تزئین**. خط، آیکون، ابر. هرگز متن.
- `sky-text` → **فقط متن**. لینک و تأکید.

## نسبت‌های کنتراست (اندازه‌گیری‌شده)

همهٔ ترکیب‌های متنیِ زیر استاندارد AA را رد می‌کنند:

| متن | روی | نسبت | |
|---|---|---|---|
| `ink` | `bg` | ۱۱٫۲:۱ | ✓ AAA |
| `ink` | `surface` | ۱۰٫۰:۱ | ✓ AAA |
| `ink` | `sky` | ۶٫۵:۱ | ✓ AA |
| `sky-text` | `bg` | ۵٫۷:۱ | ✓ AA |
| `muted` | `bg` | ۵٫۵:۱ | ✓ AA |
| `ink` | `sky-hover` | ۵٫۵:۱ | ✓ AA ← قید تنگ |
| `sky-text` | `surface` | ۵٫۱:۱ | ✓ AA |
| `muted` | `surface` | ۴٫۹:۱ | ✓ AA ← قید تنگ |
| `subtle` | `bg` | ۳٫۴:۱ | ⚠ فقط متن ≥ ۲۴px |

**دو قید تنگ که سقف روشنایی متن را تعیین می‌کنند:**

- `muted` روی کارت‌های بژ — ۴٫۹:۱. یک پرده روشن‌تر و زیر ۴٫۵ می‌رود.
- `ink` روی دکمهٔ هاور — ۵٫۵:۱. متن دکمهٔ اصلی وقتی موس رویش است.

هرگاه رنگ متن را عوض کردید، این دو را دوباره بسنجید. بقیه حاشیهٔ زیادی دارند.

> `muted` دو بار عوض شد. مقدار پیشنهادی اولیهٔ پرامپت (`#8A8178`) روی کارت‌های
> بژ فقط ۳٫۶:۱ می‌داد. مقدار قدیمی به‌عنوان `subtle` ماند، ولی فقط برای متن درشت.

## ابرها

فرم ابر (`components/brand/Cloud.tsx`) یک `path` یکپارچه است، نه چند دایرهٔ
روی هم. دلیلش کاملاً عملی است: با دایره‌های جدا، هر جا رنگ نیمه‌شفاف داده شود
(مثل ابرهای تزئینی پس‌زمینه با `text-sky-deep/10`) محل هم‌پوشانی دو برابر تیره
می‌شود و درزها بیرون می‌زند — ابر به چند لکهٔ مجزا تبدیل می‌شود.

ابرهای بزرگ صفحهٔ ورودی `sky` هستند و روی `bg` نسبت ۱٫۷:۱ دارند. برای یک شکل
پُرِ بزرگ کافی است، ولی سایهٔ نرم (`cloud-shadow`) هم کمک می‌کند فرم از زمینه
جدا شود. برچسب روی ابر `ink` است و ۶٫۵:۱ می‌دهد.

**این جدول حدس نیست.** یک ممیزی خودکار روی صفحهٔ رندرشده اجرا می‌شود که هر گره
متنی را با پس‌زمینهٔ مؤثرش (با احتساب لایه‌های نیمه‌شفاف) می‌سنجد. هر سه صفحه
صفر خطا می‌دهند.

> نکته برای هر کس که خواست دوباره اندازه بگیرد: Tailwind v4 رنگ‌ها را به‌صورت
> `oklab()` تولید می‌کند، و `canvas.fillStyle` این فرمت را نمی‌پذیرد و بی‌صدا
> مشکی برمی‌گرداند. سنجشی که این را در نظر نگیرد، همه‌چیز را مردود نشان می‌دهد.

## تایپوگرافی

فونت **استعداد**، میزبانی کاملاً لوکال در `public/fonts/` — بدون CDN.
تعریف `@font-face` در [`app/fonts.css`](app/fonts.css).

چهار وزن (۴۰۰ / ۵۰۰ / ۶۰۰ / ۷۰۰) در دو زیرمجموعه: `arabic` برای فارسی و `latin`
برای ارقام و کلمات لاتین. `unicode-range` باعث می‌شود مرورگر فقط همان چیزی را
دانلود کند که واقعاً لازم دارد. کل بار فونت: حدود ۱۶۸ کیلوبایت.

- متن معمولی: وزن ۴۰۰
- تیترها و تأکید: وزن ۵۰۰ و ۶۰۰
- فاصلهٔ خطوط باز (`leading-loose`) برای متن‌های فارسیِ طولانی

> نیم‌فاصله (`U+200C`) داخل بازهٔ عربی است — اگر روزی بازه را محدودتر کردید،
> حتماً این نقطه‌کد را نگه دارید وگرنه فاصله‌گذاری فارسی خراب می‌شود.

## حرکت

انیمیشن‌ها آرام‌اند: ترنزیشن‌ها ۳۰۰ تا ۵۰۰ میلی‌ثانیه، شناوری ابرها ۷ تا ۱۷ ثانیه.
حس آرامش اولویت است، نه جلب توجه.

هر انیمیشنی زیر `prefers-reduced-motion: reduce` خاموش می‌شود. کاربری که این
تنظیم را روشن کرده معمولاً با حرکت دچار سرگیجه می‌شود؛ سایت باید کاملاً
قابل استفاده بماند، فقط بی‌حرکت.

## قواعدی که نباید شکسته شوند

1. متن سفید روی `sky` ننویسید. پُرکنندهٔ آبی همیشه متن `ink` دارد.
2. `sky` یا `sky-deep` را متن نکنید. متنِ آبی فقط `sky-text`.
3. `subtle` را روی متن معمولی نگذارید. متن ثانویه `muted` است.
4. رنگ خام و پالت پیش‌فرض تیلویند ممنوع.
5. هر چیز کلیک‌شدنی باید `<button>` یا `<a>` واقعی باشد، نه `div` با `onClick`.
