# وب‌سایت استودیو یوگا ابر سفید

سایت **استودیو یوگا ابر سفید (White Cloud)** — باشگاه تخصصی یوگای بانوان در کرج، مهرویلا.

> ⚠️ **نسخهٔ اولیه است و آمادهٔ انتشار عمومی نیست.** بخش «کارهای باقی‌مانده» را ببینید.

## اجرا

```bash
npm install
npm run dev      # http://localhost:3000
npm run build    # خروجی استاتیک در پوشهٔ out/
```

## صفحه‌ها

| مسیر | محتوا |
|---|---|
| `/` | صفحهٔ ورودی — دو ابر + معرفی سئویی |
| `/studio/` | خانهٔ باشگاه — معرفی، مقصدها، اطلاع‌رسانی، تماس |
| `/studio/classes/` | شش سبک یوگا، با مربیان و تعداد جلسه‌های هر سبک |
| `/studio/schedule/` | برنامهٔ هفتگی، شنبه تا پنج‌شنبه |
| `/studio/teachers/` | فهرست مربیان |
| `/studio/teachers/[id]/` | صفحهٔ اختصاصی هر مربی + کلاس‌های هفتگی‌اش |
| `/studio/pricing/` | تعرفه‌ها و بسته‌های ثبت‌نام |
| `/studio/faq/` | سؤال‌های پرتکرار |
| `/shop/` | فروشگاه (اسکلت) |

## ساختار

```
app/
  page.tsx            صفحهٔ ورودی
  studio/…            صفحه‌های باشگاه
  shop/page.tsx       فروشگاه
  globals.css         توکن‌های طراحی
  fonts.css           تعریف فونت لوکال
  fonts/              فونت استعداد (woff2) — عمداً اینجا، نه در public
components/
  brand/              ابر و نشان برند
  splash/             صحنهٔ ابرهای شناور و صفحهٔ ورودی
  layout/             هدر، فوتر، سوییچ باشگاه⇄فروشگاه، پوستهٔ صفحه‌های داخلی
  ui/                 مودال، آیکون‌ها، شمارهٔ تلفن، جای‌نگهدار، پوستهٔ بخش‌ها
  studio/             Intro، News، Schedule
lib/
  nav.ts              ساختار ناوبری — هدر و نقشهٔ سایت از همین می‌خوانند
  data/               تمام محتوا — متن‌ها اینجا ویرایش می‌شوند
```

**فونت‌ها در `app/fonts/` هستند نه `public/`.** دلیلش استقرار روی زیرمسیر
است: `basePath` نشانی‌های داخل فایل CSS را بازنویسی نمی‌کند، پس با مسیر
مطلق فونت‌ها ۴۰۴ می‌شدند. از داخل `app`، باندلر خودش مسیر درست را می‌سازد.
جابه‌جایشان نکنید.

**تمام محتوا در `lib/data/` است.** برای ویرایش متن‌ها، برنامهٔ کلاس‌ها، مربی‌ها
یا تعرفه‌ها سراغ کامپوننت‌ها نروید؛ همان فایل‌های داده را عوض کنید.

**افزودن صفحهٔ جدید:** یک سطر در `lib/nav.ts` اضافه کنید تا هم‌زمان در هدر و
نقشهٔ سایت ظاهر شود — نه دو جای جدا که با هم ناهماهنگ می‌شوند.

**داده‌های مشتق هاردکد نشده‌اند.** اینکه هر سبک را کدام مربی درس می‌دهد، هر سبک
هفته‌ای چند جلسه دارد و هر مربی چه کلاس‌هایی می‌گیرد، همگی از `lib/data/schedule.ts`
محاسبه می‌شوند. برنامه را عوض کنید، این صفحه‌ها خودشان درست می‌مانند.

## طراحی

سیستم طراحی و دلیل هر تصمیم رنگی در [`DESIGN.md`](DESIGN.md) مستند شده است.

| | |
|---|---|
| فونت | استعداد — لوکال، بدون CDN |
| چیدمان | راست‌به‌چپ (RTL) |
| پالت | نوترال گرم + لهجهٔ آبی آسمانی محو |
| دسترس‌پذیری | همهٔ ترکیب‌های متنی WCAG 2.1 AA را رد می‌کنند (اندازه‌گیری‌شده روی صفحهٔ رندرشده) |

بدون هیچ وابستگی خارجی: فونت‌ها، آیکون‌ها و تصاویر همگی داخل پروژه‌اند. تنها
لینک‌های بیرونی، اینستاگرام باشگاه و مسیریابی بلد هستند — هر دو لینک قابل کلیک،
نه بارگذاری منبع.

## انتشار روی GitHub Pages

خروجی `npm run build` در پوشهٔ `out/` کاملاً استاتیک است.

- `public/.nojekyll` لازم است — بدون آن Jekyll پوشهٔ `_next/` را نادیده می‌گیرد و
  تمام CSS و JS سایت ۴۰۴ می‌شود.
- پیکربندی فعلی برای **دامنهٔ اختصاصی** (یا `username.github.io`) است.
  اگر روزی سایت زیر مسیر `username.github.io/repo-name/` منتشر شد، باید
  `basePath` و `assetPrefix` در `next.config.ts` تنظیم شود وگرنه همهٔ دارایی‌ها ۴۰۴ می‌شوند.
- برای دامنهٔ اختصاصی، فایل `public/CNAME` با نام دامنه بسازید.

## کارهای باقی‌مانده

پیش از انتشار عمومی:

- [ ] **تأیید ساعت کار** — سه منبع اینترنتی سه ساعت پایانی متفاوت می‌گویند
      (۸ شب / ۸:۳۰ / ۹ شب). فعلاً فقط «از ۷ صبح، شنبه تا پنج‌شنبه» درج شده.
- [ ] **تلفن ثابت** — شمارهٔ ۰۲۶۳۲۷۰۵۳۱۰ فقط در یک سایت دیده شد و در بیوی رسمی
      خود باشگاه نیست. عمداً درج نشده؛ پیش از افزودن تلفنی تأیید شود.
- [ ] **سبک‌های کلاس** — بیوی رسمی باشگاه «هاتا» و «ساتیاناندا» را نام می‌برد،
      ولی سایت شش سبک را فهرست کرده است. با خود باشگاه تطبیق داده شود.
- [ ] **مربیان** — نام‌ها و بیوها نمونه‌اند و باید با مربیان واقعی جایگزین شوند.
- [ ] **عکس‌ها** — همه‌جا جای‌نگهدار است. عکس واقعی فضا و مربیان اضافه شود.
- [ ] **تعرفه‌ها** — تمام قیمت‌های `lib/data/pricing.ts` نمونه‌اند و باید با نرخ
      واقعی باشگاه جایگزین شوند. شرط‌های هر بسته هم بازبینی شود.
- [ ] **سؤال‌های پرتکرار** — پاسخ‌ها بر پایهٔ رویهٔ متعارف استودیوهای یوگا نوشته
      شده‌اند؛ با رویهٔ واقعی باشگاه تطبیق داده شوند (مخصوصاً جبرانی و بارداری).
- [ ] **قیمت‌های فروشگاه** — نمونه‌اند.
- [ ] **مسیر سفارش** — دکمهٔ «سفارش» فعلاً عملکردی ندارد. محل اتصال در
      `app/shop/page.tsx` کامنت‌گذاری شده است.

## اعتبار اطلاعات

نشانی، شمارهٔ موبایل و نشانی اینستاگرام از چند منبع مستقل راستی‌آزمایی شده‌اند و
با بیوی رسمی اینستاگرام خود باشگاه تطبیق داده شده‌اند. جزئیات و منابع در
کامنت بالای [`lib/data/contact.ts`](lib/data/contact.ts) آمده است.
